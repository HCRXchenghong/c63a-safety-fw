# STM32底盘 C63A 安全冗余固件

基于 STM32F407 的 C63A 底盘控制固件，面向底盘运动控制、安全冗余、超声波感知、自动回充和上位机通信等场景开发。

当前版本已完成超声波采集侧过滤、超声波硬急停蜂鸣提示、DMA 发送健壮性修复，并通过 Debug 全量构建验证。

## 项目简介

本工程主要用于 C63A 系列底盘控制与安全冗余功能开发，核心能力包括：

- 多车型底盘运动控制
- 超声波采集与距离换算
- 超声波硬急停状态机
- 自动回充相关控制逻辑
- OLED、APP、ROS/串口、CAN 数据上报
- 蜂鸣器、RGB 灯带等本地提示功能

## 已落地功能

- 超声波采集侧过滤
- 超声波硬急停触发与保持
- 急停蜂鸣器连续鸣叫 1 秒
- ROS / 串口1 DMA 上报缓冲保护
- APP 串口 DMA 忙状态保护
- unsupported 车型 fail-safe 失能保护

## 本次关键改动

### 1. 超声波采集侧过滤

当前过滤逻辑位于 `WHEELTEC_APP/sensor_ranger.c`。

过滤规则如下：

- 测得距离 `<= 1.5 m`：保留真实值
- 测得距离 `> 1.5 m`：统一改写为 `5.1 m`
- `5.1 m`：作为当前工程中的无效值 / 超量程占位值

由于过滤发生在采集侧，所有读取 `RangerHAL_X.dis` 的模块都会直接看到过滤后的结果，包括：

- ROS / 串口上报
- OLED 显示
- RGB 灯带告警
- 自动回充逻辑
- 本地超声波硬急停逻辑

### 2. 超声波硬急停增强

当前工程已实现方向相关的超声波硬急停状态机：

- 前进时只监控前向区域
- 后退时只监控后向区域
- 原地转向或复杂运动时可进入更严格的监控模式
- 急停触发后会进入保持状态
- 车辆停稳且监控方向重新安全后才允许解除

同时，超声波硬急停触发时：

- 蜂鸣器会连续鸣叫 `1 s`
- 系统会进入失能保护
- 本地安全链路会优先阻断运动输出

### 3. DMA 发送健壮性修复

为避免 DMA 发送阶段出现栈内存被覆盖、串口忙状态重入或发送缓冲被下一帧改写的问题，当前工程已完成以下修复：

- `data_task.c` 的 ROS / 串口1 发送改为独立静态 DMA 缓冲区
- 启动 DMA 前检查串口忙状态
- `appshow_task.c` 的 APP 串口发送也改为独立 TX 缓冲区
- APP 串口发送增加 DMA 忙状态保护
- 字符串拼接改为长度受控方式，降低越界风险

### 4. 超声波超时行为修复

超声波超时分支已同步推进 `sampleSeq`，这样可以避免：

- 上一次危险结果残留
- 连续超时后 pending 状态无法正确清理
- 后续单次危险被误判为“第二次危险”

### 5. 车型安全保护

车型选择逻辑增加了 fail-safe 处理：

- 若拨码 / ADC 选择到当前固件未支持的车型
- 系统不会带着未完整初始化参数继续运行
- 会直接进入失能状态并上报错误

## 目录结构

- `Core/`：STM32Cube 生成的核心启动与外设代码
- `Drivers/`：HAL / CMSIS 驱动
- `Middlewares/`：FreeRTOS 与 USB Host 中间件
- `USB_HOST/`：USB Host 相关应用层代码
- `WHEELTEC_APP/`：底盘控制、超声波、自动回充、串口协议等应用逻辑
- `WHEELTEC_BSP/`：蜂鸣器、OLED、IMU、CAN、驱动器等板级支持代码
- `STM32CubeIDE/`：CubeIDE 工程文件与构建目录
