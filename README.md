# C63A Safety FW

基于 `STM32F407` 的 C63A 底盘控制与安全冗余固件工程。

这个仓库主要用于 C63A 系列底盘的运动控制、超声波感知、安全急停、自动回充以及上位机通信相关开发。工程当前已完成超声波采集侧过滤、硬急停蜂鸣提示、DMA 发送健壮性修复等一轮整理，并通过 `Debug` 全量构建验证。

## 项目概览

当前工程覆盖的核心能力包括：

- 底盘运动控制与多车型参数选择
- 超声波采集、距离换算与安全判定
- 超声波硬急停与失能保护
- 自动回充相关减速与控制逻辑
- OLED、APP、ROS/串口、CAN 等上报链路
- 蜂鸣器、RGB 灯带等本地提示功能

## 已落地的关键改动

### 1. 超声波采集侧过滤

当前过滤逻辑位于 `WHEELTEC_APP/sensor_ranger.c`。

过滤规则如下：

- 测得距离 `<= 1.5 m`：保留真实值
- 测得距离 `> 1.5 m`：统一改写为 `5.1 m`
- `5.1 m`：作为当前工程中的无效值 / 超量程占位值

由于过滤发生在采集侧，读取 `RangerHAL_X.dis` 的模块会直接看到过滤后的结果，包括上报、显示和本地安全逻辑。

### 2. 超声波硬急停增强

当前工程已经实现方向相关的超声波硬急停控制：

- 前进时重点监控前向区域
- 后退时重点监控后向区域
- 急停触发后会进入保持状态
- 车辆停稳且危险方向恢复安全后才允许解除

同时，超声波硬急停触发时：

- 蜂鸣器连续鸣叫 `1 s`
- 系统进入失能保护
- 本地安全链路优先阻断运动输出

### 3. DMA 发送健壮性修复

为避免 DMA 发送过程中出现缓冲区生命周期和忙状态问题，当前工程已完成以下修复：

- `data_task.c` 的 ROS / 串口1 发送改为独立静态 DMA 缓冲区
- 启动 DMA 前检查串口忙状态
- `appshow_task.c` 的 APP 串口发送增加独立 TX 缓冲区
- APP 串口发送补充 DMA 忙状态保护
- 字符串拼接改为长度受控方式，降低越界风险

### 4. 超声波超时行为修复

超声波超时分支已同步推进 `sampleSeq`，用于避免以下问题：

- 上一次危险结果残留
- 连续超时后 pending 状态无法正确清理
- 后续单次危险样本被误判为“第二次危险”

### 5. 车型选择 fail-safe 保护

车型选择逻辑已经加入安全保护：

- 若拨码 / ADC 选择到当前固件未支持的车型
- 系统不会带着未完整初始化参数继续运行
- 会直接进入失能状态并上报错误

## 仓库结构

```text
Core/             STM32Cube 生成的核心启动与外设代码
Drivers/          HAL / CMSIS 驱动
Middlewares/      FreeRTOS 与 USB Host 中间件
USB_HOST/         USB Host 相关应用层代码
WHEELTEC_APP/     底盘控制、超声波、自动回充、串口协议等应用逻辑
WHEELTEC_BSP/     蜂鸣器、OLED、IMU、CAN 等板级支持代码
STM32CubeIDE/     CubeIDE 工程文件、链接脚本与构建目录
MDK-ARM/          Keil 工程文件
```

## 开发环境

- MCU：`STM32F407`
- IDE：`STM32CubeIDE`
- Toolchain：`GNU Arm Embedded Toolchain`
- RTOS：`FreeRTOS`

2. 选择 `Debug` 配置

- 实车使用前，建议完整验证急停触发、保持与解除三个阶段
- 若修改超声波过滤逻辑，需要同时关注本地安全链路与上位机上报行为
- 提交到 Git 时，建议忽略 `Debug/Release` 和固件产物文件
